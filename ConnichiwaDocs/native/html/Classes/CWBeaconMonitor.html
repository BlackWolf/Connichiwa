Overview

Monitors the proximity for other Connichiwa beacons. If found, it reports beacons to its delegate. Only one instance of the monitor should be active at all time, therefore always use mainMonitor to get an instance of this class.

locationManager:didRangeBeacons:inRegion:

Called by CLLocationManager constantly to report found beacons in our beaconRegion. We get an array of beacons here and then check for new and changed beacons, which we then report to our delegate. Note that this method is also called with an empty beacon array

- (void)locationManager:(CLLocationManager *)manager didRangeBeacons:(NSArray *)beacons inRegion:(CLBeaconRegion *)region

Parameters

manager

The CLLocationManager instance that detected the beacons

beacons

The array of found beacons

region

The region in which the beacons where detected - should be the same as our beaconRegion
