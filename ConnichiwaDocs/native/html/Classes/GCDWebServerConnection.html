Overview

The GCDWebServerConnection class is instantiated by GCDWebServer to handle each new HTTP connection. Each instance stays alive until the connection is closed.

You cannot use this class directly, but it is made public so you can subclass it to override some hooks. Use the GCDWebServerOption_ConnectionClass option for GCDWebServer to install your custom subclass.

Warning: The GCDWebServerConnection retains the GCDWebServer until the connection is closed.

GCDWebServer internal constants and APIs.

Hooks to customize the behavior of GCDWebServer HTTP connections.

Warning: These methods can be called on any GCD thread. Be sure to also call “super” when overriding them.

overrideResponse:forRequest:

Assuming a valid HTTP request was received and either preflightRequest: or processRequest:completion: returned a non-nil GCDWebServerResponse, this method is called to override the response.

- (GCDWebServerResponse *)overrideResponse:(GCDWebServerResponse *)response forRequest:(GCDWebServerRequest *)request

Discussion

You can either modify the current response and return it, or return a completely new one.

The default implementation replaces any response matching the “ETag” or “Last-Modified-Date” header of the request by a barebone “Not-Modified” (304) one.

preflightRequest:

Assuming a valid HTTP request was received, this method is called before the request is processed.

- (GCDWebServerResponse *)preflightRequest:(GCDWebServerRequest *)request

Discussion

Return a non-nil GCDWebServerResponse to bypass the request processing entirely.

The default implementation checks for HTTP authentication if applicable and returns a barebone 401 status code response if authentication failed.
