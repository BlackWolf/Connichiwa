Namespace: Connichiwa

Overview

Connichiwa's main interface. Most of a web applications use of Connichiwa will go through this interface. It allows the web application to register for events, send messages, get information about this device and other devices and more

(static) autoConnect :Boolean

(Available on master device only)

Set to true to automatically connect nearby devices. If multiple devices with autoConnect enabled meet each other, the one that where the application was launched earlier will become the master device

Default Value:
  • false

(static) autoLoad :Array

(Available on master device only)

An array of URLs to JavaScript or CSS scripts that will be automatically loaded on any device that is connected from now on. Further, on all connected devices, the callbacks passed to Connichiwa.onLoad will only be invoked after all the scripts in this array have been loaded

Default Value:
  • [ ]

(static) broadcast(name, message, sendToSelfopt)

Broadcasts a given message to all other devices (and, if requested, also to this device). The message will have the given name and therefore trigger all callback functions that were registered for this name using Connichiwa.onMessage

(static) on(eventName, callback)

Registers a callback for a Connichiwa system event. A number of such events exist (e.g. event:devicedetected or event:deviceconnected). See the event documentation for details. These events are triggered by the framework. Registering multiple functions for the same event will invoke all those functions whenever the event is triggered

Parameters:
Name Type Description
eventName String

The name of the event to register for. Must match one of the documented system events, otherwise this method will do nothing
callback function

The callback function that will be invoked whenever the event is triggered

(static) onLoad(callback)

Register a callback to be invoked once Connichiwa is ready. The callback can be sure that the Connichiwa library is fully loaded and initialized. Also, all scripts passed to Connichiwa.autoLoad are ensured to have been loaded. Most Connichiwa-related code should be wrapped by this function. If this method is called after Connichiwa is ready, the callback will be invoked on the next run loop.

Parameters:
Name Type Description
callback function

The callback to invoke once Connichiwa is ready

(static) onMessage(name, callback)

Register a callback for a custom message. The given callback will be invoked whenever a custom message with the given name (sent using Connichiwa.send or {@link CWDevice#send}) is received on this device. The received message will be passed to the callback

Parameters:
Name Type Description
name String

The message to register for. Whenever a message with this name is received, the callback will be called

Events

onLoad

This event is fired after Connichiwa was loaded and set up properly. You can attach an event handler to this event by using the Connichiwa.onLoad method.

On remotes, onLoad will further ensure that every script added to Connichiwa.autoLoad is loaded, so your code can rely on variables and methods created in these scripts.
